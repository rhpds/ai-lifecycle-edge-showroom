= Module 1.4: Battery Monitoring System Deployment

The Battery Monitoring System (BMS) brings together all robot components to provide real-time battery health monitoring. It collects telemetry data, displays it on a dashboard, and uses the AI inference endpoints to generate predictions.

**BMS Architecture:**

* **Battery Simulator** → **MQTT Broker** → **Data Processors** → **InfluxDB** + **Dashboard**
* **Dashboard** → **Inference APIs** → **AI Predictions** → **Alerting**

[#deploy-bms-gitops]
== Task 1: Deploy BMS using GitOps

Deploy the complete Battery Monitoring System application:

[source,yaml,subs="attributes+"]
----
oc apply -f - <<EOF
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: battery-simulation
  namespace: openshift-gitops
spec:
  destination:
    server: https://kubernetes.default.svc
    namespace: battery-demo
  source:
    path: bootstrap/battery-simulation/groups/dev
    repoURL: https://github.com/rhpds/ai-lifecycle-edge-gitops.git
    targetRevision: main
    kustomize:
      patches:
        - target:
            kind: ConfigMap
            name: bms-dashboard-config
          patch: |-
            - op: replace
              path: /data/config.json
              value: |
                {
                  "BATTERY_METRICS_WS_ENDPOINT": "wss://mqtt2ws-microshift-001.{openshift_cluster_ingress_domain}/battery/metrics/",
                  "OPEN_AI_API_KEY": "my-key",
                  "STRESS_DETECTION_AI_API_ENDPOINT": "https://stress-detection-predictor-microshift-001.{openshift_cluster_ingress_domain}/",
                  "TIME_TO_FAILURE_AI_API_ENDPOINT": "https://time-to-failure-predictor-microshift-001.{openshift_cluster_ingress_domain}/",
                  "BATTERY_SIMULATION_API_ENDPOINT": "https://battery-simulation-microshift-001.{openshift_cluster_ingress_domain}/",
                  "AUTO_INFERENCE_INTERVAL_MS": 30000
                }
  project: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
EOF
----

This deploys:

* **Namespace** - `battery-demo` for BMS components
* **Battery Simulator** - Quarkus app generating realistic telemetry (voltage, temperature, distance, velocity)
* **MQTT Broker** - Mosquitto broker as central messaging hub
* **Data Processors**:
** `mqtt2ws` - Exposes data as WebSocket for dashboard
** `data-ingester` - Stores data in InfluxDB
* **InfluxDB** - Time-series database for historical sensor data
* **BMS Dashboard** - Web UI showing real-time telemetry and AI predictions

[#how-it-works]
== How the BMS works

**Data Flow:**

1. **Battery Simulator** generates realistic sensor data mimicking a robot battery in operation
2. **MQTT Broker** receives telemetry messages via MQTT protocol
3. **mqtt2ws** processor exposes data as WebSocket stream for dashboard
4. **data-ingester** processor stores data in InfluxDB for historical analysis
5. **BMS Dashboard** subscribes to WebSocket, displays real-time graphs
6. **Dashboard** periodically sends telemetry to **inference endpoints** (every 30 seconds)
7. **Inference APIs** return predictions (stress score, time-to-failure estimate)
8. **Dashboard** displays predictions and triggers alerts if thresholds are exceeded

This closed-loop system enables real-time battery health monitoring with AI-powered predictions.
