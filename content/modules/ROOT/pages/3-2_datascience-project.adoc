= Module 2.2: Review data science project

Data Science Projects in Red Hat OpenShift AI (RHOAI) organize users and AI/ML resources in a single namespace. For our factory robot battery monitoring solution, the `ai-edge-project` contains all necessary components: workbenches, data connections, model servers, and pipelines.

== Learning objectives

By the end of this module, you will be able to:

* Navigate the RHOAI Dashboard and access data science projects
* Understand how projects organize AI/ML resources in namespaces
* Review project structure including workbenches, pipelines, and models tabs
* Verify the underlying Kubernetes namespace and resources

[#access-project]
== Exercise 1: Access the ai-edge-project

1. Ensure you're in the **Red Hat OpenShift AI Dashboard**
2. Click **Data Science Projects** in the left navigation
3. Locate and click on `ai-edge-project` in the projects list

IMAGE_HERE: Data Science Projects list showing ai-edge-project

The project dashboard shows multiple tabs for managing AI/ML resources.

=== Verify

Confirm you've accessed the correct project:

✓ RHOAI Dashboard is open
✓ `ai-edge-project` appears in the Data Science Projects list
✓ Clicking the project opens the project dashboard
✓ Multiple tabs are visible (Overview, Workbenches, Pipelines, Models, Connections)

[#explore-project-structure]
== Exercise 2: Explore project structure

Review the project dashboard tabs:

* **Overview** - Project details and quick access to resources
* **Workbenches** - JupyterLab development environments
* **Pipelines** - Automated ML workflows
* **Models** - Deployed model servers for inference
* **Connections** - External storage and data sources
* **Permissions** - User access control

These tabs organize all resources for the battery monitoring AI solution.

=== Verify

Confirm all expected tabs are present:

✓ **Overview** tab - project summary
✓ **Workbenches** tab - development environments
✓ **Pipelines** tab - ML workflows
✓ **Models** tab - deployed model servers
✓ **Connections** tab - external storage links
✓ **Permissions** tab - access control

[#verify-project-namespace]
== Exercise 3: Verify the project namespace

Data Science Projects map to OpenShift namespaces. Verify the underlying namespace:

1. Switch to the **OpenShift Web Console** tab
2. Navigate to **Projects** in the left navigation
3. Find and click on the `ai-edge-project` namespace

Explore namespace resources:

**Pods:**

1. Navigate to **Workloads** → **Pods**
2. Observe running workbench and model server pods

**Services:**

1. Navigate to **Networking** → **Services**
2. View model inference endpoints

**Storage:**

1. Navigate to **Storage** → **PersistentVolumeClaims**
2. See data storage volumes for workbenches

The namespace contains all Kubernetes resources for the AI/ML project.

=== Verify

Confirm the project namespace exists with expected resources:

**In OpenShift Console:**

✓ `ai-edge-project` namespace exists in Projects list
✓ **Workloads → Pods**: Multiple running pods (workbench, model servers)
✓ **Networking → Services**: Model inference endpoints visible
✓ **Storage → PVCs**: Persistent volumes for workbenches

**Example resources you should see:**
- `model-training-xxxxx` pod (JupyterLab workbench)
- `stress-detection-predictor-xxxxx` pod (model server)
- `time-to-failure-predictor-xxxxx` pod (model server)
- `model-training` PVC (workbench storage)
