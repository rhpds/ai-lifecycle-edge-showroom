= Module 3.3: Train Battery Health Models

Execute the training notebooks sequentially to train Stress Detection and Time-to-Failure models. The notebooks automate data retrieval, preprocessing, training, validation, and model upload.

[#understanding-training-workflow]
== Understanding the Training Workflow

The training process consists of **seven notebooks** executed in sequence:

1. **01_retrieve-influx-data.ipynb** - Fetch battery telemetry from InfluxDB, save locally
2. **02_prepare-data.ipynb** - Preprocess and normalize data for training
3. **03_bms-stress-training.ipynb** - Train Stress Detection model, save locally
4. **04_predict-stress.ipynb** - Validate Stress Detection model with test data
5. **05_bms-ttf-training.ipynb** - Train Time to Failure model, save locally
6. **06_predict-ttf.ipynb** - Validate Time to Failure model with test data
7. **07_save-models.ipynb** - Upload trained models to robot's MinIO storage

[#what-happens-during-training]
== What happens during execution

As you run the notebooks:

**1. Data Retrieval** - Fetches battery telemetry (voltage, temperature, charge cycles) from InfluxDB on robots

**2. Data Preparation** - Cleans, normalizes, and splits data into training/validation/test sets

**3. Stress Detection Training** - Neural network learns battery stress patterns
   * Input: 9 features (voltage, temp, SoC, discharge rate, cycles, etc.)
   * Output: Stress score (0-1, threshold: 0.5)

**4. Stress Model Validation** - Tests accuracy against held-out data

**5. Time-to-Failure Training** - Regression model predicts remaining battery life
   * Input: 5 features (temp delta, voltage delta, resistance, current delta, capacity fade)
   * Output: Estimated hours until failure

**6. TTF Model Validation** - Evaluates prediction accuracy

**7. Model Upload** - Saves models to MinIO in OpenVINO format
   * Path: `stress-detection/1/` and `time-to-failure/1/`
   * Format: `.xml` (model structure) and `.bin` (weights)
