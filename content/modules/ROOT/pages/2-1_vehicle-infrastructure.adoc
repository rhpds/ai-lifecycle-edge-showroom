= Module 1.1: Robot infrastructure setup

This module explores the infrastructure running on our factory transportation robots. Each robot runs Red Hat Device Edge (MicroShift), a lightweight Kubernetes platform optimized for resource-constrained environments.

== Learning objectives

By the end of this module, you will be able to:

* Connect to factory robot infrastructure via bastion host and SSH
* Verify MicroShift is running and operational on edge devices
* Explore deployed namespaces and understand GitOps readiness
* Understand MicroShift's minimal footprint and suitability for edge deployments

[#connect-robot]
== Exercise 1: Connect to the factory robot

Access the robot infrastructure via the bastion host, then connect to the MicroShift virtual machine.

=== Connect to bastion host

Establish an SSH connection to the bastion host:

[source,bash,subs="attributes+"]
----
{bastion_ssh_command}
----

**Password:** `{bastion_ssh_password}`

=== Connect to robot VM

Once connected to the bastion, access the factory robot's MicroShift VM:

[source,bash,subs="attributes+"]
----
virtctl ssh cloud-user@vmi/microshift -n microshift-001 --identity-file=/home/lab-user/.ssh/microshift-001
----

=== Verify

Confirm you are connected to the MicroShift VM:

[source,bash]
----
hostname
----

Expected output:
----
microshift
----

✓ Hostname shows `microshift`

✓ You are now inside the factory robot's VM

[#verify-microshift]
== Exercise 2: Verify MicroShift is running

MicroShift is a lightweight Kubernetes distribution designed for edge devices. It provides:

* **Minimal footprint** - Runs with ~2 CPU cores and 2GB RAM
* **Kubernetes API** - Full compatibility with standard Kubernetes tools
* **Edge-optimized** - Built for resource-constrained, distributed deployments
* **Embedded capabilities** - Model serving and GitOps built-in

Check that MicroShift is running:

[source,bash]
----
sudo systemctl status microshift
----

Expected output should show `active (running)`.

Verify cluster access:

[source,bash]
----
oc get nodes
----

You should see one node in `Ready` status.

=== Verify

Confirm MicroShift is operational:

**Expected systemctl output:**
----
● microshift.service - MicroShift
     Loaded: loaded (/usr/lib/systemd/system/microshift.service; enabled)
     Active: active (running) since ...
----

**Expected nodes output:**
----
NAME                    STATUS   ROLES                         AGE   VERSION
localhost.localdomain   Ready    control-plane,master,worker   5d    v1.32.6
----

✓ MicroShift service shows `active (running)`

✓ Node status is `Ready`

✓ Kubernetes API is accessible via `oc` command

[#explore-namespaces]
== Exercise 3: Explore deployed namespaces

List the namespaces already deployed on the robot:

[source,bash]
----
oc get namespaces
----

You should see several namespaces including:

* `openshift-gitops` - GitOps packages in MicroShift
* `redhat-ods-applications` - Model serving packages in MicroShift

NOTE: Some namespaces may not exist yet. You'll deploy these components using GitOps in the following sections.
