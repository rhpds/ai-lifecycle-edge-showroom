[#review-datasciencecluster]
== Task 3: Review DataScienceCluster Configuration

The DataScienceCluster custom resource defines which RHOAI components are enabled and how they're configured.

1. In **Installed Operators**, click **Red Hat OpenShift AI**
2. Navigate to the **DataScienceCluster** tab
3. Click the existing DataScienceCluster instance
4. Click the **YAML** tab to view the configuration

=== Understanding key components

**KServe Configuration:**

1. Locate the `kserve` section in the YAML
2. Note: `defaultDeploymentMode: RawDeployment`

**Why RawDeployment for Edge?**

* Full control over Kubernetes resources without Knative overhead
* Ideal for resource-constrained SNO environments
* Predictable resource usage (no auto-scaling)
* Simpler configuration for edge deployments

3. Verify: `managementState: Managed`

**KNative-Serving Configuration:**

1. Locate the `serving` section (KNative-Serving stack)
2. Note: `managementState: Removed`

**Why Remove KNative?**

* RawDeployment doesn't use KNative
* Reduces resource consumption on SNO
* Eliminates unnecessary components

**TrustyAI Configuration:**

1. Locate the `trustyai` section
2. Verify: `managementState: Managed`

**Why Enable TrustyAI?**

* Monitors model bias and fairness
* Detects data drift over time
* Essential for automated retraining (models continuously update without human oversight)

You can also view component status visually:

1. Click the **Details** tab
2. Scroll to see component configurations
