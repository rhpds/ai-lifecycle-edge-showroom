[#review-dataconnection]
== Exercise 2.3: Review DataConnection

DataConnections link RHOAI to external object storage. The `vehicle-models` connection provides access to the robot's MinIO storage where AI models and training data are stored.

**DataConnections enable:**

* **Workbenches** - Access training data from robot storage
* **Model Servers** - Load models for validation
* **Pipelines** - Read input data and write trained models

The connection bridges the SNO training environment and robot's edge storage.

=== Review the vehicle-models DataConnection

. Ensure you're in the **RHOAI Dashboard** inside the `ai-edge-project` project
. Navigate to the **Connections** tab
. Locate the existing connection: `vehicle-models`
. Click on the **three vertical dots** on the right and click **Edit**
+
image::04-module-02_connection.png[Connections tab showing vehicle-models DataConnection,link=self,window=blank,width=700]

=== Understand connection parameters

The `vehicle-models` connection is configured with:

* **Connection type**: S3 compatible object storage - v1 (standard S3 protocol compatible with MinIO)
* **Connection name**: `vehicle-models`
* **Endpoint**: `\http://minio-microshift-vm.microshift-001.svc.cluster.local:30000` (Robot's MinIO storage)
* **Region**: `us-east-1` (default S3 region, required but unused by MinIO)
* **Bucket**: `inference` (contains AI models in the robot)

=== Verify the underlying secret

DataConnections are stored as Kubernetes Secrets. The secret contains base64-encoded S3 credentials:

* `AWS_ACCESS_KEY_ID` - MinIO username
* `AWS_SECRET_ACCESS_KEY` - MinIO password
* `AWS_S3_ENDPOINT` - MinIO API endpoint
* `AWS_DEFAULT_REGION` - S3 region
* `AWS_S3_BUCKET` - Target bucket name

NOTE: All values are base64-encoded for security. RHOAI components automatically decode them when using the connection.

=== Verify

Confirm the DataConnection is properly configured:

✓ `vehicle-models` connection appears in Connections tab

✓ **Endpoint**: Points to MicroShift MinIO service

✓ **Bucket**: `inference`
