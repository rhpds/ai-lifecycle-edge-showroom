= Module 1.2: MinIO storage deployment

MinIO provides S3-compatible object storage for AI models on the factory robot. The `inference` bucket stores two AI models: **Stress Detection** (identifies battery stress conditions) and **Time to Failure** (predicts remaining battery life).

Both models are stored in OpenVINO IR format (`.xml` and `.bin` files) in versioned directories:

* `/stress-detection/1/` - Stress Detection model v1
* `/time-to-failure/1/` - Time to Failure model v1

== Learning objectives

By the end of this module, you will be able to:

* Deploy MinIO object storage using GitOps automation
* Understand S3-compatible storage architecture for edge AI models
* Verify model files are properly stored in versioned directories
* Access and navigate the MinIO web dashboard

[#deploy-minio-gitops]
== Exercise 1: Deploy MinIO using GitOps

GitOps automates deployment and ensures consistency across edge devices. Changes to the Git repository automatically sync to the robot.

=== Create GitOps application project

Create the default project for GitOps applications:

[source,yaml]
----
oc apply -f - <<EOF
kind: AppProject
apiVersion: argoproj.io/v1alpha1
metadata:
  name: default
  namespace: openshift-gitops
spec:
  clusterResourceWhitelist:
  - group: '*'
    kind: '*'
  destinations:
  - namespace: '*'
    server: '*'
  sourceRepos:
  - '*'
EOF
----

=== Create MinIO GitOps application

Deploy MinIO by creating a GitOps Application:

[source,yaml]
----
oc apply -f - <<EOF
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: minio-microshift
  namespace: openshift-gitops
spec:
  destination:
    server: https://kubernetes.default.svc
  source:
    path: bootstrap/minio-microshift/groups/dev
    repoURL: https://github.com/rhpds/ai-lifecycle-edge-gitops.git
    targetRevision: main
  project: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
EOF
----

This deploys:

* **Namespace** - `minio` namespace for isolation
* **Deployment** - MinIO server instance
* **Service & Route** - Web dashboard access
* **PersistentVolumeClaim** - Storage for model files
* **Bucket creation** - Automatically creates `inference` bucket
* **Model download** - Downloads baseline Stress Detection and Time to Failure models from GitHub

=== Verify

Confirm the GitOps Application was created:

[source,bash]
----
oc get application -n openshift-gitops minio-microshift
----

Expected output:
----
NAME               SYNC STATUS   HEALTH STATUS
minio-microshift   Synced        Healthy
----

✓ Application exists in `openshift-gitops` namespace

✓ Sync status shows `Synced`

✓ Health status shows `Healthy` (may take 1-2 minutes)

[#monitor-deployment]
== Exercise 2: Monitor MinIO deployment

Watch the deployment progress:

[source,bash]
----
watch oc get pods -n minio
----

Press `Ctrl+C` to exit when pods show `Running` and `Completed` status.

=== Verify

Check pod status:

[source,bash]
----
oc get pods -n minio
----

Expected output:
----
NAME                                READY   STATUS      RESTARTS   AGE
create-bucket-job-xxxxx             0/1     Completed   0          2m
download-models-job-xxxxx           0/1     Completed   0          2m
minio-xxxxx-xxxxx                   1/1     Running     0          2m
----

✓ `minio` pod shows `Running` with `1/1` ready

✓ `create-bucket-job` shows `Completed` status

✓ `download-models-job` shows `Completed` status

[#access-minio-dashboard]
== Exercise 3: Access MinIO dashboard

Once deployed, access the MinIO web console:

[source,bash,subs="attributes+"]
----
https://minio-microshift-001.{openshift_cluster_ingress_domain}
----

**Credentials:**

* Username: `minio`
* Password: `minio123`

Verify the `inference` bucket contains two directories:

* `stress-detection/1/` - Baseline Stress Detection model files (`.xml`, `.bin`)
* `time-to-failure/1/` - Baseline Time to Failure model files (`.xml`, `.bin`)

image::2-2_minio-inference.png[MinIO dashboard showing inference bucket with model directories,link=self,window=blank,width=700]

=== Verify

Navigate through the MinIO dashboard:

1. Click on the **inference** bucket
2. Verify you see two folders: `stress-detection/` and `time-to-failure/`
3. Click into `stress-detection/1/` - verify you see `.xml` and `.bin` files
4. Go back and click into `time-to-failure/1/` - verify you see `.xml` and `.bin` files

Expected files in each model directory:
----
stress-detection/1/
  ├── model.xml          (OpenVINO model definition)
  └── model.bin          (OpenVINO model weights)

time-to-failure/1/
  ├── model.xml
  └── model.bin
----

✓ `inference` bucket exists

✓ Both model directories present

✓ Each directory contains `.xml` and `.bin` files
