= Module 5.3: Schedule pipeline automation

Scheduled pipelines enable fully automated model retraining. Configure the pipeline to run every 10 minutes, ensuring models continuously improve without manual intervention.

== Learning objectives

By the end of this module, you will be able to:

* Create a scheduled pipeline run with periodic triggers
* Configure pipeline schedules (every 10 minutes for continuous retraining)
* Monitor automated pipeline executions over time
* Understand how scheduling enables self-improving edge AI systems

[#why-scheduled-automation]
== Why scheduled automation matters

Machine learning models drift over time as operational conditions change:

* **Battery aging** - Performance degrades as batteries wear
* **Temperature variations** - Seasonal changes affect battery behavior
* **Workload patterns** - Different operational routes stress batteries differently

**Scheduled Retraining Benefits:**

* **Eliminates manual intervention** - No human involvement required
* **Reduces model drift** - Models stay accurate with fresh data
* **Ensures reliability** - Predictive accuracy maintained over time
* **Enables true edge AI** - Autonomous systems that self-improve

For our 18-robot fleet operating 24/7, automation is essential.

[#understand-automated-workflow]
== Understanding the Automated Workflow

Every 10 minutes, the scheduled pipeline:

1. **Retrieves** - Fresh battery telemetry from factory robots (last hour of data)
2. **Processes** - Normalizes and prepares training datasets
3. **Trains** - New Stress Detection and Time-to-Failure models
4. **Validates** - Compares new models against existing models
5. **Deploys** - Uploads improved models to robot storage (if they outperform)
6. **Repeats** - Cycle continues every 10 minutes

This creates a **self-improving system** where models get better over time without human intervention.
