= Module 4.2: Review time to failure server

The Time to Failure server predicts the remaining operational life of robot batteries. Like the Stress Detection server, it validates newly trained models before deployment.

== Learning objectives

By the end of this module, you will be able to:

* Review Time-to-Failure model server configuration
* Compare two complementary model servers (Stress vs TTF)
* Verify separate model paths in shared storage
* Understand how multiple models use the same ServingRuntime

[#review-ttf-endpoint]
== Exercise 3: Review the model endpoint

In the Time to Failure details, locate **Internal endpoint details**.

The endpoint format:

[source,text]
----
http://time-to-failure-<namespace>.svc.cluster.local/v2/models/time-to-failure/infer
----

Applications can call both endpoints to get comprehensive battery health insights:

* **Stress Detection** - Is the battery currently under stress?
* **Time to Failure** - How much longer will the battery last?
