= Module 6.1: Enable TrustyAI Monitoring

Automated model retraining enables continuous improvement but introduces uncertainty. Models update every 10 minutes without human oversight. TrustyAI provides continuous monitoring to ensure models remain reliable, fair, and accurate.

[#why-model-monitoring]
== Why model monitoring matters

**The Challenge:**

* **Automated retraining** - New models deployed every 10 minutes
* **No human review** - Models update without data scientist approval
* **Potential issues** - Bias, drift, degraded performance

**TrustyAI Solution:**

* **Bias detection** - Identifies unfair patterns in predictions
* **Data drift** - Detects changes in input distribution
* **Model performance** - Tracks accuracy over time

For factory robots where safety and efficiency depend on predictions, monitoring is essential.

[#enable-user-workload-monitoring]
== Task 2: Enable User Workload Monitoring

TrustyAI collects metrics using Prometheus. Enable user workload monitoring:

**Create User Workload Monitoring ConfigMap:**

Click the **+** icon (Import YAML) in the OpenShift Console:

[source,yaml]
----
oc apply -f - <<EOF
apiVersion: v1
kind: ConfigMap
metadata:
  name: user-workload-monitoring-config
  namespace: openshift-user-workload-monitoring
data:
  config.yaml: |
    prometheus:
      logLevel: debug
      retention: 15d
EOF
----

Click **Create**.

**Enable Cluster Monitoring:**

Click the **+** icon again:

[source,yaml]
----
oc apply -f - <<EOF
apiVersion: v1
kind: ConfigMap
metadata:
  name: cluster-monitoring-config
  namespace: openshift-monitoring
data:
  config.yaml: |
    enableUserWorkload: true
EOF
----

Click **Create**.

[#deploy-trustyai-service]
== Task 3: Deploy TrustyAI Service

Create the TrustyAI service in the `ai-edge-project` namespace.

Click the **+** icon (Import YAML):

[source,yaml]
----
oc apply -f - <<EOF
apiVersion: trustyai.opendatahub.io/v1alpha1
kind: TrustyAIService
metadata:
  name: trustyai-service
  namespace: ai-edge-project
spec:
  storage:
    format: "PVC"
    folder: "/inputs"
    size: "1Gi"
  data:
    filename: "data.csv"
    format: "CSV"
  metrics:
    schedule: "5s"
    batchSize: 5000
EOF
----

Click **Create**.

**Understanding the Configuration:**

* **Storage** - 1GB PVC for inference data and metrics
* **Data format** - CSV for model inputs and outputs
* **Metrics schedule** - Calculate metrics every 5 seconds
* **Batch size** - Process up to 5000 inferences per batch
