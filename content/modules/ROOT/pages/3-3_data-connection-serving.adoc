= Module 2.3: Review data connection

DataConnections link Red Hat OpenShift AI (RHOAI) to external object storage. The `vehicle-models` connection provides access to the robot's MinIO storage where AI models and training data are stored.

== Learning objectives

By the end of this module, you will be able to:

* Understand how DataConnections link RHOAI to external storage
* Review the `vehicle-models` S3 connection configuration
* Verify the underlying Kubernetes Secret containing S3 credentials
* Explain how connections bridge SNO training environment and robot storage

[#understanding-dataconnections]
== Understanding DataConnections

DataConnections are stored as OpenShift Secrets containing S3 credentials. They enable:

* **Workbenches** - Access training data from robot storage
* **Model Servers** - Load models for validation
* **Pipelines** - Read input data and write trained models

The connection bridges the SNO training environment and robot's edge storage.

[#understand-connection-parameters]
== Exercise 2: Understand connection parameters

The `vehicle-models` connection is configured with:

* **Connection type**: S3 compatible object storage - v1 (standard S3 protocol compatible with MinIO)
* **Connection name**: `vehicle-models`
* **Endpoint**: `http://minio-microshift-vm.microshift-001.svc.cluster.local:30000`
* **Region**: `us-east-1` (default S3 region, required but unused by MinIO)
* **Bucket**: `inference` (contains AI models and training data)

**Endpoint Explanation:**

The URL `minio-microshift-vm.microshift-001.svc.cluster.local:30000` uses Kubernetes service DNS to connect across clusters. The `microshift-001.svc.cluster.local` domain allows SNO to communicate with the robot's MicroShift cluster over the factory network.

=== Verify

✓ **Connection name**: `vehicle-models`
✓ **Endpoint**: Points to MicroShift MinIO service
✓ **Bucket**: `inference`
✓ **Region**: `us-east-1` (standard default)

[#verify-underlying-secret]
== Exercise 3: Verify the underlying secret

DataConnections are stored as Kubernetes Secrets. Review the secret:

1. Switch to the **OpenShift Web Console**
2. Navigate to **Workloads** → **Secrets**
3. Ensure you're in the `ai-edge-project` namespace (use the project dropdown)
4. Search for or scroll to: `aws-connection-vehicle-models`
5. Click on the secret to view details

The secret contains base64-encoded S3 credentials:

* `AWS_ACCESS_KEY_ID` - MinIO username
* `AWS_SECRET_ACCESS_KEY` - MinIO password
* `AWS_S3_ENDPOINT` - MinIO API endpoint
* `AWS_DEFAULT_REGION` - S3 region
* `AWS_S3_BUCKET` - Target bucket name

NOTE: All values are base64-encoded for security. RHOAI components automatically decode them when using the connection.
