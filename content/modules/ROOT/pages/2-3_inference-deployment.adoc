= Module 1.3: Model serving deployment

The inference server loads AI models from MinIO and exposes REST API endpoints for real-time predictions. It uses KServe (part of Red Hat OpenShift AI) optimized for edge deployments.

**Why Model Serving Matters:**

* **Real-time predictions** - Low-latency inference for battery monitoring
* **Model management** - Loads models from MinIO storage automatically
* **API endpoints** - REST APIs for the Battery Monitoring System
* **Resource efficiency** - Optimized for resource-constrained edge environments

== Learning objectives

By the end of this module, you will be able to:

* Deploy KServe model serving infrastructure using GitOps
* Understand ServingRuntime and InferenceService custom resources
* Configure model servers to load models from S3-compatible storage
* Test inference endpoints using KServe v2 protocol

[#model-serving-architecture]
== Understanding model serving on MicroShift

The model serving infrastructure consists of:

**ServingRuntime:** Defines the model server container (OpenVINO Model Server) that supports multiple formats:

* OpenVINO IR (used for our models)
* ONNX
* TensorFlow
* PyTorch
* PaddlePaddle

**InferenceService:** Connects models from MinIO to the ServingRuntime:

* References the S3 bucket path (e.g., `s3://inference/stress-detection/`)
* Configures network access (Service and Route)
* Manages model lifecycle (loading, versioning, scaling)

**Service Account:** Provides S3 credentials to access MinIO storage

**Routes:** Expose inference endpoints externally for API access

[#deploy-inference-gitops]
== Exercise 1: Deploy model serving using GitOps

Deploy the inference server GitOps application:

[source,yaml]
----
oc apply -f - <<EOF
apiVersion: argoproj.io/v1alpha1
kind: Application
metadata:
  name: model-server
  namespace: openshift-gitops
spec:
  destination:
    server: https://kubernetes.default.svc
  source:
    path: bootstrap/model-server/groups/dev
    repoURL: https://github.com/rhpds/ai-lifecycle-edge-gitops.git
    targetRevision: main
  project: default
  syncPolicy:
    automated:
      prune: true
      selfHeal: true
EOF
----

This deploys:

* **Namespace** - `inference` namespace for model serving
* **ServingRuntime** - OpenVINO Model Server with REST and gRPC support
* **Service & Routes** - External access to inference endpoints
* **Service Account** - S3 access permissions for MinIO
* **InferenceServices** (2):
** `stress-detection` - Loads stress detection model from MinIO
** `time-to-failure` - Loads time-to-failure model from MinIO

=== Verify

Confirm the GitOps Application was created:

[source,bash]
----
oc get application -n openshift-gitops model-server
----

Expected output:
----
NAME           SYNC STATUS   HEALTH STATUS
model-server   Synced        Healthy
----

✓ Application exists in `openshift-gitops` namespace
✓ Sync status shows `Synced`
✓ Health status shows `Healthy`

[#test-inference-endpoints]
== Exercise 3: Test inference endpoints

Verify that models are loaded and responding to inference requests.

=== Understanding inference POST requests

Inference requests use the KServe v2 REST API format:

**Endpoint:** `POST http://<pod-ip>:8888/v2/models/<model-name>/infer`

**Request Body Structure:**

[source,json]
----
{
  "inputs": [
    {
      "name": "keras_tensor",       // Input tensor name (from model definition)
      "shape": [1, 9],               // Batch size 1, 9 features
      "datatype": "FP32",            // 32-bit floating point
      "data": [0.5, 0.5, ...]       // Normalized sensor values
    }
  ]
}
----

**Input Features (Stress Detection - 9 values):**

* Voltage, Temperature, State of Charge, Discharge Rate, Time Elapsed, Cycle Count, Ambient Temperature, Usage Pattern, Health Status

**Input Features (Time to Failure - 5 values):**

* Temperature Delta, Voltage Delta, Resistance, Current Delta, Capacity Fade

**Response Structure:**

[source,json]
----
{
  "outputs": [
    {
      "name": "output",
      "shape": [1, 1],
      "datatype": "FP32",
      "data": [0.87]              // Prediction score or time estimate
    }
  ]
}
----

=== Test stress detection model

Send a test inference request:

[source,bash]
----
curl -s -X POST http://$(oc get pod -n inference -l app=isvc.stress-detection-predictor -o jsonpath='{.items[0].status.podIP}'):8888/v2/models/stress-detection/infer \
     -H "Content-Type: application/json" \
     -d '{"inputs": [{"name": "keras_tensor", "shape": [1, 9], "datatype": "FP32", "data": [0.5, 0.5, 0.5, 0.5, 0.5, 0.5, 0.5, 0.5, 0.5]}]}' | \
     jq -r '.outputs[0].data[0] | if . > 0.5 then "STRESSED" else "NORMAL" end'
----

Expected output: `STRESSED` or `NORMAL`

=== Test time to failure model

Send a test inference request:

[source,bash]
----
curl -s -X POST http://$(oc get pod -n inference -l app=isvc.time-to-failure-predictor -o jsonpath='{.items[0].status.podIP}'):8888/v2/models/time-to-failure/infer \
     -H "Content-Type: application/json" \
     -d '{"inputs": [{"name": "keras_tensor", "shape": [1, 5], "datatype": "FP32", "data": [40.0, 48.0, 162.5, -2.0, -18.0]}]}' | \
     jq -r '.outputs[0].data[0] | "Predicted Time Before Failure: \(.) hours"'
----

Expected output: `Predicted Time Before Failure: <number> hours`
