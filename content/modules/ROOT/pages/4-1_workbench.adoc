= Module 3.1: Review Model Training Workbench

A workbench is a containerized JupyterLab environment with pre-installed AI/ML frameworks. The `model-training` workbench provides everything needed to develop and train battery health prediction models.

[#understanding-workbenches]
== Understanding workbenches

Workbenches provide:

* **JupyterLab** - Interactive development environment
* **Pre-installed frameworks** - TensorFlow, PyTorch, scikit-learn, etc.
* **Data connections** - Direct access to robot storage via `vehicle-models`
* **Persistent storage** - Notebooks and datasets survive pod restarts
* **GPU support** - When available for training acceleration

2. Click the **Workbenches** tab

[#understand-workbench-configuration]
== Task 2: Understand Configuration Choices

**Image Selection:**

The `Jupyter | TensorFlow | CUDA | Python 3.12` image includes:

* **TensorFlow** - Neural network framework for training models
* **CUDA support** - GPU acceleration (if hardware available)
* **Python 3.12** - Latest Python with data science libraries (NumPy, Pandas, Matplotlib)
* **JupyterLab** - Interactive notebook interface

TIP: Click **View package information** next to any image to see the complete list of pre-installed packages.

**Persistent Storage:**

The workbench has a persistent volume: `model-training-storage` (20GB)

This storage persists:

* Jupyter notebooks
* Training datasets
* Model checkpoints
* Experiment results

NOTE: Even if the workbench pod restarts, all data remains intact.

**Data Connection:**

Expand the **Connections** section. Verify `vehicle-models` is attached.

This connection enables:

* Reading training data from robot's MinIO
* Accessing existing model files
* Saving retrained models back to robot storage

=== Verify

Confirm workbench configuration details:

✓ **Image** includes TensorFlow, CUDA support, Python 3.12
✓ **Persistent Storage** shows `model-training-storage` (20GB)

[#review-workbench-openshift]
== Task 3: Review Workbench Resources in OpenShift

Switch to the **OpenShift Web Console** to examine underlying resources.

**Workbench Pod:**

1. Navigate to **Workloads** → **Pods**
2. Ensure you're in the `ai-edge-project` namespace
3. Find the pod starting with `model-training` (e.g., `model-training-0`)
4. Click the pod to view details:
   * **Status**: Running
   * **Containers**: Jupyter notebook server
   * **Volumes**: `model-training-storage` PVC mounted
   * **Environment**: Variables from `vehicle-models` connection

**Persistent Storage:**

1. Navigate to **Storage** → **PersistentVolumeClaims**
2. Find and click `model-training-storage`
3. Review PVC details:
   * **Capacity**: 20GB
   * **Access mode**: ReadWriteOnce
   * **Status**: Bound

=== Verify

**Expected PVC:**

✓ Workbench pod shows `2/2` ready and `Running` status
