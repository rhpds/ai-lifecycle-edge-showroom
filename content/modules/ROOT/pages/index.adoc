[#about-this-lab]
== About this workshop

This hands-on workshop demonstrates an end-to-end AI solution for battery health monitoring in factory transportation robots. You'll explore Red Hat Device Edge (MicroShift), Single Node OpenShift (SNO), and Red Hat OpenShift AI working together to enable intelligent automation at the edge.

The lab covers pre-configured infrastructure, automated model training pipelines, model serving, and monitoring, demonstrating how AI models can continuously improve without manual intervention.

This workshop is designed for **platform engineers, ML engineers, DevOps teams, and data scientists** working with:

* AI/ML workloads at the edge with limited resources
* Automated MLOps pipelines for continuous model improvement
* Kubernetes-based edge infrastructure (MicroShift, Single Node OpenShift)
* Red Hat OpenShift AI for the full ML lifecycle
* IoT, industrial automation, or edge AI use cases

[#scenario]
== The scenario

An automotive parts distribution facility operates **18 battery-powered autonomous mobile robots (AMRs)** that transport components 24/7. Unexpected battery failures cause production delays, disrupted schedules, and safety risks.

**The Solution:** AI-powered battery health monitoring

* **On each robot (MicroShift)** - Real-time telemetry collection, AI inference, alerting
* **At the plant (SNO + RHOAI)** - Automated model retraining every 10 minutes with fresh data
