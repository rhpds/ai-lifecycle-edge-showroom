[#run-training]
== Exercise 3.3: Execute training notebooks

Run the training notebooks to train both battery health prediction models. The notebooks demonstrate the complete ML workflow from data collection to model export.

=== Understand the training workflow

Both notebooks follow the same pattern:

1. **Data Collection** - Retrieve battery telemetry from InfluxDB on robots
2. **Data Preprocessing** - Clean, normalize, and prepare training datasets
3. **Model Training** - Train neural network models using TensorFlow
4. **Model Evaluation** - Validate performance on test data
5. **Model Export** - Convert to OpenVINO IR format for edge deployment
6. **Model Upload** - Store in MinIO for distribution to robots

**Stress Detection Model:**

* **Input**: 9 features (voltage, temperature, state of charge, discharge rate, time elapsed, cycle count, ambient temperature, usage pattern, health status)
* **Output**: Binary classification (STRESSED vs NORMAL, threshold 0.5)
* **Architecture**: Neural network with dense layers and dropout

**Time to Failure Model:**

* **Input**: 5 features (temperature delta, voltage delta, resistance, current delta, capacity fade)
* **Output**: Regression (predicted hours until failure)
* **Architecture**: Neural network optimized for time-series prediction

=== Run Stress Detection training

. Open `stress-detection-training.ipynb`
. Execute cells sequentially using **Shift+Enter** or **Run** → **Run All Cells**
. Monitor execution:
+
* **Data retrieval** - Fetches telemetry from InfluxDB
* **Preprocessing** - Normalizes features, creates train/test split
* **Training** - Trains model, shows loss/accuracy metrics
* **Evaluation** - Tests on validation data, displays confusion matrix
* **Export** - Converts to OpenVINO IR format (`.xml` and `.bin` files)
* **Upload** - Stores model in MinIO `inference/stress-detection/1/` bucket

IMAGE_HERE: Jupyter notebook showing Stress Detection training with loss curves and accuracy metrics

Wait for all cells to complete execution. Training typically takes 2-5 minutes.

[#troubleshooting]
== Troubleshooting

**Issue**: Workbench fails to start or shows "Spawner timeout"

**Solution**:

. Check workbench pod status: `oc get pods -n ai-edge-project | grep model-training`
. Verify pod is running: `oc describe pod <workbench-pod> -n ai-edge-project`
. Check resource quotas: `oc get resourcequota -n ai-edge-project`
. Common fix: Stop and restart the workbench from RHOAI Dashboard

**Issue**: Git clone fails with "Authentication required" error

**Solution**:

. Verify repository URL is correct (public repositories don't need auth)
. Check network connectivity from workbench pod
. Try cloning via terminal: `git clone https://github.com/rhpds/ai-lifecycle-edge-gitops.git`
. Common fix: Use HTTPS URL instead of SSH for public repos

**Issue**: Notebook execution fails with "ModuleNotFoundError"

**Solution**:

. Verify Python kernel is selected (top-right of notebook)
. Check installed packages: Run `!pip list` in a notebook cell
. Install missing packages: `!pip install <package-name>`
. Common fix: Restart kernel and run all cells again

**Issue**: InfluxDB data retrieval returns empty dataset

**Solution**:

. Verify InfluxDB is accessible from workbench
. Check InfluxDB credentials in notebook configuration
. Verify battery simulation is generating data on robot
. Check InfluxDB query time range includes recent data
. Common fix: Adjust query time range to "Past 24 hours"

**Issue**: Model upload to MinIO fails with "Access Denied"

**Solution**:

. Verify DataConnection credentials are configured in project
. Check MinIO accessibility: `curl http://minio-microshift-vm.microshift-001.svc.cluster.local:30000`
. Verify bucket exists: Check MinIO dashboard
. Common fix: Recreate DataConnection with correct credentials
