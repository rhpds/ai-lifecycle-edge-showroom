= Module 3.2: Import Training Notebooks

Pre-built Jupyter notebooks contain all the code for training Stress Detection and Time-to-Failure models. Import them from a GitHub repository to accelerate development.

[#clone-repository]
== Task 1: Clone the Training Repository

In JupyterLab, clone the repository containing training notebooks:

1. Locate the toolbar on the left side of JupyterLab
2. Click the **Git** icon (third position)

IMAGE_HERE: JupyterLab sidebar showing Git icon

3. In the Git menu, click **Clone repository**
4. Enter the repository URL:

[source,text]
----
https://github.com/rhpds/ai-lifecycle-edge-gitops.git
----

5. Click **Clone**

The repository clones into the `ai-lifecycle-edge-gitops` folder, visible in the file browser.

IMAGE_HERE: JupyterLab file browser showing cloned ai-lifecycle-edge-gitops folder structure

=== Verify

Confirm the repository was cloned successfully:

✓ Git clone operation completes without errors
✓ `ai-lifecycle-edge-gitops` folder appears in file browser
✓ Folder contains subdirectories (notebooks, bootstrap, etc.)
✓ Git icon in sidebar shows repository status

[#explore-repository-contents]
== Task 2: Explore Repository Contents

The cloned repository contains:

* **Notebooks** - Jupyter notebooks for training Stress Detection and Time-to-Failure models
* **Data** - Sample battery telemetry data from factory robots
* **Scripts** - Helper scripts for model validation and deployment
* **Pipelines** - Pipeline definitions for automated retraining workflows

Navigate to `ai-lifecycle-edge-gitops/notebooks/training/` to see the training notebooks.

=== Verify

Confirm repository contents are accessible:

Expected directory structure:
----
ai-lifecycle-edge-gitops/
├── bootstrap/          (GitOps deployment manifests)
├── notebooks/
│   ├── training/       (Training notebooks - we'll use these)
│   └── pipelines/      (Pipeline definitions)
├── scripts/            (Helper utilities)
└── README.md
----

✓ Navigate to `notebooks/training/` folder successfully
✓ Folder contains multiple `.ipynb` notebook files
✓ Notebooks for Stress Detection and Time-to-Failure models are present
✓ All repository files are readable
